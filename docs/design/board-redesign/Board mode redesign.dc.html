The reasoning — arguments strong enough to touch the Settled list
1Resting collapses to a rollupOPEN 1

Settled says Resting is never meant to be read. So its default state should cost one line per project, not a 233-card scroll. Collapsing it doesn't fight "quietest, narrowest column" — it delivers it. Grouped by project because that's the only axis you'd ever search a resting thread on; "Show all" expands to newest-first, capped groups. Try the Resting header — it toggles.

2The void was a side-effect, not the shapeOPEN 2

Three short columns looked broken only because a 233-tall column stretched the board beside them. Remove that column (via #1), top-align the rest, and the empties collapse to one calm band. Columns are kept — the alternative (rows) breaks "one glance tells me what needs me." Space below the band is the correct resting state of a tool whose best day is nothing needs me.

3Move-to anchors down, ⋯ on hover, Archive backOPEN 3 · 8B

The trigger dodged corner controls only because it was persistent. Make it hover-revealed and it has no permanent footprint — so it can sit in the one place that reads as "more," inner-top-right, and the popover anchors to it and opens downward (auto-flips near the viewport edge), never over the header. Three targets + divider + Archive restored. Working stays out: it's an observed fact, not a judgement.

4One card, treatment driven per columnOPEN 4

Keep a single card component; let per-column tokens decide how loud it is. Needs Help maxes the question — 14px, italic, full contrast, amber frame — and the inferred variant subordinates it (dashed, dimmed, ~ marker, dismiss). Resting drops to one 55%-opacity line. Same anatomy, opposite volume: honest about what the board knows.

6Filter chips: one row + dropdownOPEN 6

Only projects with non-resting activity earn a chip; the rest live behind "All projects ▾". A chip is a filter, and filtering to an all-resting project yields an empty board — so it earns no space above the fold. The strip stays one row, scrolls horizontally with an edge fade. Twenty projects never becomes two rows.

8The ✓✓ bug, designed outOPEN 8A

The column owns the ✓. Before rendering, strip any leading check glyph (✓ ✔ ☑ done:) from the agent's own summary, then prepend exactly one. The tick becomes a status affordance the card controls, not text the agent can duplicate.

7Counts live above the sections

Section headers can be collapsed, so counts can't only live there — the whole point (see that two threads need you without a gesture) would break. A persistent four-count strip sits above the stack; the header counts are a repeat for the sections you've scrolled to. Swipe is a shortcut on Complete only, never the way to discover what needs you.

Swipe = acknowledge, on Complete only

Acknowledgement is the one safe destructive-feeling action (it's reversible, and real activity un-clears it). Swipe-right reveals the ✓. Needs Help keeps its explicit Answer button — you never want to clear an unanswered ask by accident.

Resting grows the same way it does on desktop

Collapsed by default to its count; tapping expands to the grouped rollup. It never contributes to the scroll you do to reach Working.
