# Dispatch v2.8.6 — Pretty-mode polish: compaction, command echoes, staged images

## The change

A batch of fixes to Pretty mode (the rendered chat for Claude and Codex threads),
the CLI ⇄ Pretty switch, and the mobile projects list.

- **Compaction now reads as compaction.** While a thread is compacting its
  context, the chat showed the same **"Working…"** row as an in-flight answer — so
  a message sent mid-compaction (which just queues) looked like a reply on the way.
  It now shows a distinct **"Compacting context…"** indicator, and catches
  auto-compaction, not just a manual `/compact`.
- **Slash-command echoes stop masquerading as your messages.** Running `/compact`
  wrote `<local-command-stdout>Compacted</local-command-stdout>` as a green user
  bubble of raw XML. Those echoes now demote to a muted one-line notice (or drop
  when they carry no text), the same treatment background-task notifications get.
- **Paste a screenshot, then caption it.** Pasted or attached images used to fire
  as their own turn the instant they landed. They now stage in the composer as
  removable thumbnails and send with your typed message as one turn.
- **The resume-advice card lost its clutter** — no more accent left border, no
  "Don't ask again".
- **One render switch, and only between turns.** The frontend-only View/Terminal
  toggle rendered a CLI thread as a fake-pretty transcript, parallel to and
  confusable with real Pretty. It's gone: a CLI thread shows the real terminal,
  and CLI ⇄ Pretty is the single switch — now disabled mid-turn, since switching
  then kills the process out from under a streaming response.
- **The mobile projects list can sort.** Recent / Alphabetical / Custom, sharing
  the same preference as the desktop sidebar. And the ⇅ sort glyph now matches the
  `+` button's weight and size instead of sitting thin and small beside it.

## Under the hood

- Compaction was already tracked (`system/status: compacting`) but only reached
  the corner context indicator; the prominent busy row keyed on `busy` alone.
  `ChatView` now lets `compacting` win that slot.
- Command-echo detection is a content-shape classifier (`classifyUserText`),
  duplicated core-side and web-side like the task-notification twin, anchored to
  the whole message so a human quoting a tag keeps their bubble. It also guards the
  resume-anchor scan so a resume never anchors on "Compacted".
- Staged images mirror the coordinator Composer's `composerImages` buffer, folded
  into `doSend` as a `[text, ...images]` content array.
- Removing View mode deleted `ConversationView`, `ModeToggle`, `useStructuredStream`
  and the `threadMode` store; `TabHost` and the board's `WorkerLightbox` render a
  CLI thread as `TerminalTab`. The transport switch gates on the live `threadStatus`.
- The mobile projects sort reuses `SortMenu` and mirrors `ProjectSidebar`'s
  comparator; the glyph derives a responsive Arial font matching the `+` per
  breakpoint. This is a web change — a browser refresh surfaces it — while the
  compaction/command-echo/resume-anchor fixes touch the daemon and need a restart.
