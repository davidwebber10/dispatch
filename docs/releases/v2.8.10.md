# Dispatch v2.8.10 — Desktop `+` and sort icons render again

## The change

On desktop, the **add (`+`)** and **sort** icons in a project card's Threads/Automations
header had collapsed into a 4-pixel sliver — effectively invisible. They're back to full
size.

## What was wrong

The recent icon-consolidation swapped the old font-glyph `+`/sort marks for Phosphor
`<svg>` icons. Those icon buttons set an explicit `width: 16px` but no padding — so they
inherited the browser's **default `<button>` padding** (~6px each side). Combined with the
app's global `* { box-sizing: border-box }`, that padding is subtracted *from* the 16px,
leaving only a **4px** content box. A text glyph simply overflowed that box and looked
fine; an SVG is a flex item, so it shrank to the 4px width — a thin vertical sliver (13px
tall, 4px wide).

Mobile was unaffected because its icon buttons are larger (34px → 22px of content, plenty
for the icon), which is why the same controls looked correct on a phone.

## The fix

One line: the shared icon-button style now sets `padding: 0`, giving the 13px icon its full
16px box back. This covers every `+` and sort control in the project card at once.

Verified against the live DOM: 24 squished icons → 0, each now rendering at its full size.

This is a **web-only** change — a browser refresh (or the auto-update) is all it takes; no
daemon rebuild/restart required.
