# doppler-mcp

A stdio [Model Context Protocol](https://modelcontextprotocol.io) server that exposes [Doppler](https://www.doppler.com) secret-management tools (`doppler_list_secrets`, `doppler_get_secret`, and — unless read-only — `doppler_set_secret` and `doppler_delete_secret`) to MCP clients such as Claude and Codex, talking to the Doppler REST API on your behalf. It is configured entirely through environment variables: `DOPPLER_TOKEN` (required — a Doppler service/CLI token; the process exits if it is missing), `DOPPLER_PROJECT` and `DOPPLER_CONFIG` (optional defaults used when a tool call omits `project`/`config`), and `DOPPLER_READ_ONLY=1` (optional — when set, the write tools are not registered). It speaks JSON-RPC over stdio, so it is launched as a subprocess by the host (e.g. an MCP `command`/`args` entry in your Claude or Codex config pointing at the built `dist/index.js` binary, with the `DOPPLER_*` variables supplied in that entry's `env`); all diagnostics go to stderr so stdout stays a clean protocol channel.
